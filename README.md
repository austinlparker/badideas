# badideas

a home for projects that are bad ideas

## the dream website

> "That's my dream website. PHP executed in the client via WASM, connecting to
> a WASM MySQL server. Pulls a prebuilt DB dump from github.io. Time to first
> contentful paint measured in minutes."

It is built. It is real. Every word of the requirement is honored.

### What happens when you visit

1. Your browser downloads **`mysqld.wasm`** — an 84MB MySQL-compatible server
   ([go-mysql-server](https://github.com/dolthub/go-mysql-server), reporting
   itself as MySQL 8.0.31) compiled from Go to WebAssembly — and boots it. It
   listens on nothing, in your RAM.
2. Your browser fetches **`dump.sql`**, a prebuilt mysqldump-style database
   dump, from github.io, and replays it into the server **one statement at a
   time**.
3. Your browser then downloads a **PHP 8.4 interpreter**
   ([php-wasm](https://github.com/seanmorris/php-wasm)) and boots that too.
4. Your browser fetches **`index.php`** as plain text — the thing every
   webserver config since 1995 has existed to prevent — and executes it. The
   PHP queries MySQL through a synchronous JavaScript bridge, one
   JSON-serialized call at a time, increments a hit counter (a real `UPDATE`),
   and renders a guestbook.
5. Only then does anything appear on screen. Nothing is parallelized, nothing
   is streamed, and the page stays intentionally blank until PHP finishes —
   a loading spinner would imply apology. Progress is reported to the
   developer console, where all honest progress lives.

There is no backend. Every visitor ships their own database server, making
this the most horizontally scaled MySQL deployment in history. Signing the
guestbook performs a real `INSERT` into a real database that evaporates when
you close the tab.

### Layout

| path | crime |
|---|---|
| `site/index.html` | the loader; orchestrates the whole catastrophe, sequentially |
| `site/index.php` | a 2003-vintage PHP guestbook, executed by your browser |
| `site/dump.sql` | the prebuilt DB dump (generated by `tools/generate_dump.py`) |
| `mysqld-wasm/` | Go wrapper that compiles go-mysql-server to WASM and exposes `mysqlQuery()` to JS |
| `tools/vendor-php-wasm.sh` | vendors the php-wasm interpreter into the site |
| `.github/workflows/pages.yml` | builds all of the above and deploys to GitHub Pages |

### PHP → MySQL, without a socket

php-wasm ships the `vrzno` extension, which lets PHP call JavaScript
synchronously. The Go side registers a global `mysqlQuery(sql)` returning
JSON. So the driver is:

```php
$window = new Vrzno;
$res = json_decode($window->mysqlQuery('SELECT hits FROM hit_counter'), true);
```

mysqli walked so this could crawl.

### Building locally

```bash
./mysqld-wasm/build.sh        # needs Go >= 1.24; emits site/mysqld.wasm (84MB)
./tools/vendor-php-wasm.sh    # needs Node; vendors PHP into site/php-wasm/
cd site && python3 -m http.server 8000
# open http://localhost:8000 and go make coffee
```

(One caveat inside `build.sh`: vitess references `syscall.SIGHUP`, which does
not exist on `GOOS=js` because browsers famously do not deliver POSIX signals
to tabs. The build vendors the deps and rewrites it to raw signal 1, which
compiles fine and will simply never fire.)

### Deploying

Enable GitHub Pages for this repo with **Settings → Pages → Source: GitHub
Actions**. Every push to `main` compiles the MySQL server, vendors the PHP
interpreter, and deploys the lot to github.io.

### Performance

Time to first contentful paint scales linearly with your distance from
GitHub's CDN and your tolerance for consequences. On localhost it paints in
about 8 seconds, which we consider a bug.
